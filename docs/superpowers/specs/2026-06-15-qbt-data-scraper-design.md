# QBT 控盘数据采集 Chrome 扩展 — 设计文档

## 概述

Chrome 浏览器扩展，用户在弹窗中输入品牌列表和目标 URL，扩展自动逐个品牌检索 Nint 数据，分别获取"全部"和"淘宝全部"两个渠道的数据，最终导出为 CSV 文件。

## 架构

```
Popup (popup.html/js)  ←→  Content Script (content.js)
        ↑                        ↑
        └── Background (background.js, Service Worker)
```

- **Popup**：用户交互入口，输入品牌列表和 URL，控制采集流程，展示进度，触发 CSV 导出
- **Content Script**：注入目标页面，执行 DOM 操控（选择下拉框、输入品牌、点击复选框、点击检索、解析表格）
- **Background**：轻量 Service Worker，桥接 Popup 和 Content Script 的消息通信

## 自动化流程（每个品牌）

1. 检查下拉框是否选中"自定义" → 不是则点击并选择"自定义"
2. 输入品牌名称到搜索框
3. 确保"全部"已选中（未选中则点击）→ 点击"检索" → 等待页面加载 → 解析表格
4. 取消"全部" → 选中"淘宝全部" → 点击"检索" → 等待页面加载 → 解析表格
5. 返回两份数据

## 表格解析规则

- 移除第2行（销量子表头）
- 移除最后一行（总计行）
- 移除最后一列（总计列）
- 在最前面插入品牌名称列

各元素的 XPath 硬编码在 content script 中。

## CSV 输出格式

列：渠道, 品牌名称, 类别名称, 月份1, 月份2, ...

每行 = 一个品牌在一个渠道下的一个价格段数据

## 关键决策

- 使用 Manifest V3
- 弹窗内粘贴文本输入品牌（一行一个）
- 支持自定义目标 URL
- 全自动顺序处理，弹窗展示进度
- CSV 文件下载导出
