# QBT 自定义价格段控盘取数

Chrome 浏览器扩展，自动从 Nint 平台批量检索品牌数据，支持多渠道对比、多价格带和 CSV 导出。

## 功能

### 核心采集
- **批量品牌采集**：输入品牌列表，自动逐个检索
- **双布局自适应**：自动识别页面有无"全部"复选框，适配不同查看范围布局
  - 布局 A（有全部）：全部 → 淘宝全部 → 计算天猫
  - 布局 B（无全部）：淘宝全部 → 天猫全部 → 计算全部
- **多价格带支持**：自动识别价格带数量，每个价格带独立输出一行
- **多维度支持**：月维度（202604）和日维度（2026-01-01）均可正常采集
- **智能跳过**：品牌无数据或与基线一致时自动标记失败，不写入 CSV
- **基线对比**：采集前先空白检索获取基线数据，品牌数据与基线一致则判定取数失败
- **自动完成**：输入品牌名后调用接口自动选择下拉列表第一项

### CSV 导出
- **完整字段**：类目 > 渠道 > 品牌名称 > 价格带 > 日期数据
- **类目自动提取**：从页面面包屑自动获取多级类目（一级>二级>三级）
- **渠道自动计算**：全部 = 淘宝全部 + 天猫全部（或天猫 = 全部 − 淘宝全部）
- BOM 头支持，Excel 直接打开无需转码

### 用户体验
- **逐品牌状态**：采集时 textarea 替换为品牌状态列表，实时展示每个品牌的成功/失败/无数据
- **进度持久化**：关闭弹窗再打开，进度和状态自动恢复
- **重置按钮**：一键清空品牌列表和采集进度
- **快捷键**：Enter 开始采集
- **Excel 粘贴**：支持直接粘贴品牌列（Tab 分隔自动拆分）

## 安装

1. 下载代码或 `git clone https://github.com/aidenLi377/QBT_Brand-Channel.git`
2. 打开 Chrome，访问 `chrome://extensions`
3. 开启右上角"开发者模式"
4. 点击"加载已解压的扩展程序"
5. 选择项目目录

## 使用

1. 打开 Nint 目标页面（`art.nint.com/stat-ali-new?...`）
2. 点击扩展图标，在弹窗中输入品牌列表（一行一个，支持 Excel 粘贴）
3. 点击"开始采集"或按 Enter
4. 等待采集完成，查看品牌状态列表确认结果
5. 点击"导出 CSV"

### CSV 输出格式

| 类目 | 渠道 | 品牌名称 | 价格带 | 2026-01-01 | ... |
|------|------|---------|--------|------------|-----|
| 美容>面部精华 | 淘宝全部 | Dior/迪奥 | 200-∞元 | 246972 | ... |
| 美容>面部精华 | 淘宝全部 | Dior/迪奥 | 100-199.99元 | 19708 | ... |
| 美容>面部精华 | 天猫全部 | Dior/迪奥 | 200-∞元 | ... | ... |
| 美容>面部精华 | 全部 | Dior/迪奥 | 200-∞元 | ... | ... |
